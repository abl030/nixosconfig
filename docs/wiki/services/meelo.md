# Meelo

Date researched: 2026-05-13
Status: PostgreSQL extracted to nspawn on `proxmox-vm`
Related: issue #230, #228, #232

## Service Shape

Meelo runs on `proxmox-vm` with state under `/mnt/virtio/meelo` and is exposed at `https://meelo.ablz.au/` through the module in `modules/nixos/services/meelo.nix`.

Most of the stack remains upstream OCI containers:

- `meelo-server`
- `meelo-scanner`
- `meelo-matcher`
- `meelo-front`
- `meelo-mq`
- `meelo-search`
- `meelo-nginx`
- `meelo-transcoder`

The database is the exception. Issue #230 narrowed the work to extracting PostgreSQL from the bundled OCI database container into the fleet `mk-pg-container` pattern. Non-database image pinning and moving RabbitMQ or Meilisearch to nixpkgs services are explicitly out of scope.

Current database runtime:

- Unit: `container@meelo-db.service`
- Address: `192.168.100.17:5432`
- PostgreSQL: 14.22 via `pkgs.postgresql_14`
- Data directory: `/mnt/virtio/meelo/postgres-nspawn/postgres`
- Database/user: `meelo` / `meelo`

Meelo's server image runs Prisma migrations at startup. The `meelo` database role therefore has `CREATEDB` inside this isolated database container so Prisma can create its shadow database. Do not grant broader host or cross-service database access for this.

- Source container: `meelo-db`

- Source PostgreSQL version: `14.1`
- Source database/user: `meelo` / `meelo`

The target nspawn database uses PostgreSQL 14 so the move is not also an unplanned major-version upgrade. The target data parent is separate from the old OCI data directory:

- Old OCI data: `/mnt/virtio/meelo/postgres`
- New nspawn parent: `/mnt/virtio/meelo/postgres-nspawn`
- New nspawn data: `/mnt/virtio/meelo/postgres-nspawn/postgres`

Keep the old OCI data directory intact until the migrated runtime is verified and the rollback window is deliberately closed.

## Secret Layout

`secrets/meelo.env` is the broad app secret file used by the non-database Meelo stack. It no longer contains PostgreSQL or `DATABASE_URL` values.

`secrets/hosts/proxmox-vm/meelo-pgpass.env` is the narrow PostgreSQL secret file used by:

- `mk-pg-container` for `POSTGRES_PASSWORD`
- `meelo-server` for `DATABASE_URL`
- `meelo-transcoder` for libpq-style `PG*` variables

The pgpass secret is loaded after `meelo/env` for the server. After the migration, the non-database containers (`meelo-scanner`, `meelo-matcher`, `meelo-mq`, `meelo-search`) were restarted and verified not to expose `POSTGRES*`, `PG*`, or `DATABASE_URL` variables.

## Rollback While Old OCI Data Is Retained

Rollback is available while `/mnt/virtio/meelo/postgres` remains intact:

1. Restore the previous module shape with the bundled `meelo-db` OCI container.
2. Restore the old database variables to `secrets/meelo.env` or adapt the rollback module to use `secrets/hosts/proxmox-vm/meelo-pgpass.env`.
3. Rebuild `proxmox-vm`.
4. Verify `podman-meelo-db.service`, `podman-meelo-server.service`, and `podman-meelo-nginx.service`.
5. Check `https://meelo.ablz.au/`.

Do not delete or move `/mnt/virtio/meelo/postgres` until the migration has soaked and rollback is no longer needed.
